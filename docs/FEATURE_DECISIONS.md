# Feature Decisions

## Відмова від Electron

Рішення: не продовжувати Electron-версію як основний напрямок.

Причина:

* веб-сайт простіше публікувати;
* легше отримувати органічний трафік;
* простіше оновлювати продукт;
* нижчий поріг входу для лікарів.

## Перехід на Next.js

Рішення: новий продукт реалізовано на Next.js + React + Tailwind CSS.

Причина:

* зручний деплой на Vercel;
* добра основа для SEO;
* можливість у майбутньому додавати окремі сторінки;
* проста структура для MVP.

## Назва

Рішення: використовувати назву "Асистент лікаря".

Причина:

* ширше за кардіологію;
* дозволяє масштабувати продукт на інші спеціальності;
* зрозуміло для сімейних лікарів, терапевтів, кардіологів та інтернів.

## OpenAI відкладено

Рішення: не підключати OpenAI API на цьому етапі.

Причина:

* спочатку потрібна стабільна форма та продуктова логіка;
* треба окремо вирішити питання безпеки медичних даних;
* поточний SEO-етап не потребує AI.

## Авторизація відкладена

Рішення: не додавати авторизацію.

Причина:

* MVP має бути максимально простим;
* немає серверного збереження медичних даних;
* авторизація ускладнить продукт без негайної користі.

## База даних відкладена

Рішення: не додавати базу даних.

Причина:

* поточний продукт працює як відкритий інструмент;
* немає потреби зберігати пацієнтів на сервері;
* це зменшує ризики щодо персональних медичних даних.

## Обрана дизайн-концепція

Рішення: концепція №2 — сучасний SaaS для лікарів.

Візуальний напрям:

* Clinical Blue;
* SaaS-акуратність;
* багато повітря;
* чисті картки;
* стриманий професійний вигляд.

## Обраний логотип

Рішення: Modular A.

Ідея:

* абстрактна літера A;
* модульність;
* технологічність;
* без медичних кліше.

Заборонені символи:

* червоний хрест;
* серце;
* стетоскоп;
* чаша зі змією;
* лікар у халаті.
