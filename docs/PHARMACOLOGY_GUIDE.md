# Правила модуля «Препарати»

## Призначення

Вкладка «Препарати» — практичний довідник для лікаря. Вона допомагає швидко порівняти препарати за клінічно важливими відмінностями, але не замінює офіційну інструкцію, клінічні настанови або рішення лікаря.

Назва вкладки в інтерфейсі: **«Препарати»**. Внутрішні компоненти можуть використовувати назву `Pharmacology`.

## Структура

* дані кожного класу зберігати окремо у `data/drugs/`;
* спільний інтерфейс будувати через компоненти у `components/pharmacology/`;
* нову групу додавати як окремий розділ усередині вкладки, без нової верхньої вкладки;
* не додавати окремі SEO-сторінки без прямого запиту;
* дані працюють локально, без API, авторизації та бази даних.

## Формат картки

Картка препарату має бути компактною та згортаною. У згорнутому стані показувати назву, звичайну дозу, кратність і період напіввиведення. У розгорнутому стані залишати тільки інформацію, корисну для вибору між препаратами:

* міжнародна та українська назви;
* 2–3 приклади актуальних торгових назв в Україні;
* звичайна і максимальна дози;
* кратність прийому;
* період напіввиведення, якщо це практично важливо;
* орієнтовний ефект у цифрах лише за наявності надійних досліджень;
* 2–3 короткі речення «Що відрізняє препарат».

Не додавати латинську назву, довгу інструкцію, порівняльну таблицю або однакові застереження в кожну картку.

## Медичний зміст

* не називати препарат «найкращим» без чітких критеріїв і доказів;
* не подавати середній ефект як гарантований результат для пацієнта;
* вказувати, для якої дози, показання та способу вимірювання наведено числовий орієнтир;
* не змішувати дозування для артеріальної гіпертензії, серцевої недостатності та інших показань;
* спільні протипоказання, моніторинг і правила безпеки показувати один раз на рівні класу;
* не формувати персоналізоване призначення або автоматичну схему лікування.

## Джерела

Перед публікацією перевірити:

1. Дози, показання і застереження — за чинною офіційною інструкцією або авторитетною клінічною настановою.
2. Числовий ефект — за систематичним оглядом, метааналізом або якісним порівняльним дослідженням.
3. Торгові назви — за Державним реєстром лікарських засобів України.
4. Фактичну присутність на ринку — додатково за актуальними українськими аптечними каталогами.

Реєстрація препарату не означає, що він зараз продається. Не використовувати іноземну торгову назву, якщо її присутність в Україні не підтверджена. Для монопрепарату не вказувати назву фіксованої комбінації.

## UX

* використовувати один загальний пошук для всіх класів; пошук має працювати за міжнародною, українською та торговими назвами;
* класи препаратів показувати як accordion; одночасно розгортати не більше одного класу;
* одночасно розгортати не більше однієї картки;
* не перевантажувати вкладку вкладеними картками та довгими текстами;
* спільний блок безпеки розміщувати один раз у згортаних клінічних примітках класу;
* загальний медичний дисклеймер розміщувати один раз унизу вкладки;
* перевіряти desktop і mobile після кожного розширення.

## Поточні рішення для БРА

* порівняльної таблиці немає;
* латинські назви не показуються;
* числове зниження САТ є лише орієнтиром із досліджень;
* торгові назви мають відповідати українському ринку;
* для валсартану використовувати «Діокор Соло», а не «Діокор», оскільки «Діокор» є комбінацією з гідрохлортіазидом;
* для кандесартану вказані «Касарк» і «Кантаб»;
* для лозартану «Козаар» не використовувати за рішенням власника продукту.

## Поточні рішення для ІАПФ

* перший набір: еналаприл, лізиноприл, раміприл, периндоприл і каптоприл;
* період напіввиведення показувати для кожного препарату, а для проліків уточнювати активний метаболіт;
* не створювати рейтинг «найкращий ІАПФ при ХХН», оскільки настанови рекомендують клас ІАПФ або БРА без визначення однієї найкращої молекули;
* правила застосування при ХХН і спільні ризики показувати один раз на рівні класу;
* «Каптопрес» не використовувати як торгову назву каптоприлу, оскільки це комбінація з гідрохлоротіазидом;
* для периндоприлу обов'язково розрізняти дози трет-бутиламінової та аргінінової солей.
* не повторювати однакове орієнтовне зниження АТ у кожній картці ІАПФ; класовий ефект і посилання на Cochrane-огляд показувати один раз у клінічних примітках класу;
* для каптоприлу пояснювати, що швидший ранній ефект не є доказом більшої антигіпертензивної сили протягом доби.

## Перевірка перед завершенням

1. Перевірити медичні джерела та українські торгові назви.
2. Переконатися, що текст пояснює відмінності, а не повторює загальні властивості класу.
3. Запустити `npm run build`.
4. Перевірити пошук, accordion і мобільне відображення у браузері.
5. Оновити `docs/CURRENT_STATE.md`.
6. Після успішної перевірки зробити commit, push і production deployment відповідно до правил проєкту.
